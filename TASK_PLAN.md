# План выполнения задач MMSS

## Задача MMSS_003: Реализация TextQ-in-Area-Creator: Живая инструментальная среда для контекстного управления смыслами
* **Статус:** `DONE`
* **Комментарии:** Эта задача завершает переход от статичного оркестратора к живой семантической среде. Использовать существующие стили из \templates\orchestrator.html.

### План разрешения:

### I. Подготовка и Базовая Интеграция
1.  **Обновление `orchestrator_v2.html`:**
    *   Добавить HTML-структуру для модального окна "Создать инструмент".
    *   Добавить HTML-структуру для панели "MemoryTools" (для пресетов).
    *   Убедиться, что `orchestrator_v2.html` включает `orchestrator_main.js` и `orchestrator_v2.js`.
2.  **Инициализация в `orchestrator_v2.js`:**
    *   Создать базовую структуру для управления инструментами и состоянием сессии.
    *   Инициализировать пустой массив для хранения инструментов.

### II. Создание Инструмента (Frontend)
3.  **Обработка выделения текста:**
    *   В `orchestrator_v2.js`, добавить обработчик события `mouseup` или `selectionchange` для `semantic-node` или `answer-panel`.
    *   При выделении текста, отображать +кнопку "Создать инструмент" рядом с выделением.
4.  **Модальное окно "Создать инструмент":**
    *   При нажатии на +кнопку, отображать модальное окно с полями "Название действия" (например, "Максимально детализировать") и "Текст промпта" (например, 'Детализируй: {{selected_text}}').
    *   Поле "Текст промпта" должно автоматически заполняться шаблоном, например, 'Детализируй: {{selected_text}}'.
    *   Добавить чекбокс "Сохранить как пресет".
5.  **Сохранение инструмента:**
    *   При подтверждении в модальном окне, сохранять новый инструмент (название, промпт, является ли пресетом) в локальное хранилище (localStorage) и/или в текущее состояние сессии.

### III. Применение Инструмента (Frontend & Backend Interaction)
6.  **Отображение кнопок инструментов:**
    *   Под каждым `semantic-node`, динамически отображать созданные инструменты (как кнопки).
    *   Если инструмент является пресетом, он должен быть доступен глобально или через панель "MemoryTools".
7.  **Обработка нажатия кнопки инструмента:**
    *   При нажатии на кнопку инструмента, извлекать `selected_text` (если есть) и подставлять его в `Текст промпта` инструмента.
    *   Инициировать запрос к MMSS-агенту через `/orchestrate/send` (используя существующую логику `sendAgentRequest` из `orchestrator_main.js`).
    *   Полученный ответ отображать как дочерний `semantic-node`.

### IV. Управление Пресетами и Сессиями
8.  **Кнопка "Restore MemoryTools":**
    *   В `orchestrator_v2.html`, добавить кнопку "Restore MemoryTools".
    *   В `orchestrator_v2.js`, при нажатии на эту кнопку, загружать все сохранённые пресеты из localStorage и отображать их.
9.  **Фрактальное хранение состояния сессии:**
    *   Модифицировать логику сохранения/загрузки проекта (`save_orchestrator_project` в `app.py` и соответствующий JS на фронтенде) для включения состояния Area-Creator (созданные инструменты, их привязка к узлам, история ветвлений).
    *   Каждый `semantic-node` должен иметь `data-branch-id` для отслеживания иерархии.

### V. Семантическая Разметка (helpLinks-forAI)
10. **Обработка тегов `<span class="help-link" ...>`:**
    *   В `orchestrator_v2.js`, добавить логику для рендеринга этих тегов.
    *   При наведении на `help-link`, отображать тултип с информацией из `data-mmss-system` и `data-metric`.

### VI. Orchestrator-дирижёр
11. **Кнопка "Создать сводный ответ":**
    *   В `orchestrator_v2.html`, добавить кнопку "Создать сводный ответ".
    *   В `orchestrator_v2.js`, при нажатии на эту кнопку:
        *   Собрать все пользовательские вопросы/инструменты из текущей сессии (фрактального дерева).
        *   Сформировать структурированный промпт для `OMNIAGENT_UNIОтFIED_SYNTHESIZER`.
        *   Отправить запрос через `/orchestrate/send`.
        *   Отобразить ответ как новый `semantic-node`.

### VII. Тестирование и Валидация
12. **Тестирование функциональности:**
    *   Проверить создание, сохранение и применение инструментов.
    *   Проверить работу пресетов и их восстановление.
    *   Проверить фрактальное ветвление и сохранение состояния.
    *   Проверить семантическую разметку.
    *   Проверить работу "Orchestrator-дирижёра".
