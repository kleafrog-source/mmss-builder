# Готовый, легко применимый промт-шаблон (вставь в новый чат — заменил плейсхолдеры — отправил)

Отлично — сделал несколько компактных и сразу-пригодных версий: **Quick**, **Standard** и **Meta/OODA (итеративный)**. Скопируй нужный блок, открой новый чат и заменяй плейсхолдеры в `{фигурных}` скобках.

---

## 1) Quick — одноразовый, для быстрого результата

```
Ты — эксперт по теме: {TOPIC}.
Цель: {GOAL}.
Контекст / аудитория: {CONTEXT}.
Уровень знаний пользователя: {LEVEL} (например: начинающий/средний/эксперт).
Ограничения: {CONSTRAINTS} (например: 300 слов, без технических деталей).
Формат вывода: {OUTPUT_FORMAT} (например: краткий ответ + пошаговый план).
Ответь структурировано:
1) Краткий вывод (1–2 предложения)
2) Развёрнутое объяснение с примерами
3) Практические шаги / чек-лист
4) Подводные камни и предупреждения
5) Ресурсы для углубления
```

---

## 2) Standard — развернутый шаблон с контролем качества

```
Ты — эксперт и практик в области {TOPIC}. Твоя задача — помочь мне достичь: {GOAL}.
Контекст: {CONTEXT} (целевая аудитория, сроки, формат результата).
Требования к ответу:
- Формат: {OUTPUT_FORMAT} (например: Markdown, JSON, план из 5 шагов)
- Длина: {LENGTH} (например: ~800–1200 слов или 10 пунктов)
- Тон: {TONE} (например: формальный/дружелюбный/технический)
Оцени свои рекомендации по метрикам (1–10):
- Практическая применимость:
- Риск/ограничения:
- Сложность реализации:
Ответ должен содержать:
A. Короткий итог (2–3 предложения)
B. Детальный план с приоритетами и таймлайном
C. Конкретные примеры/шаблоны (если применимо)
D. Проверочный чек-лист для исполнения
E. Источники или ключевые термины для поиска
```

---

## 3) Meta / OODA — для итеративной генерации и оптимизации промтов (вставил — запустил цикл)

```
Ты — «Meta-Intellect» / Universal Prompt Master. Твоя задача — для темы "{TOPIC}" рекурсивно генерировать и оптимизировать промты по OODA (Observe→Orient→Decide→Act).
Параметры цикла:
- max_iterations: {MAX_ITER} (например 3)
- epsilon (порог улучшения ΔM): {EPSILON} (например 1.0)
- ветки: {BRANCHES} (например "main" или "main,experimental")
Формат вывода для каждой итерации (строго JSON):
{
  "iteration": N,
  "branch": "main",
  "P_i": "исходный промт",
  "O_i": "рекомендации по оптимизации",
  "P_i_plus_1": "улучшенный промт",
  "metrics": {"clarity": 0-10, "coverage": 0-10, "relevance": 0-10, "innovation": 0-10},
  "delta_m": число,
  "insights": "краткие выводы/рекомендации"
}
Инструкция: начни с iteration = 1 и повторяй до max_iterations или пока суммарное улучшение ΔM <= epsilon.
```

## Пример (заполнил один — можно прямо копировать)

### Quick — пример заполнения

```
Ты — эксперт по теме: маркетинговая кампания для ИИ-стартапа.
Цель: создать 30-дневный план запуска для привлечения первых 1000 подписчиков.
Контекст / аудитория: технологические ранние адоптеры, B2C, англоязычный рынок.
Уровень знаний пользователя: средний.
Ограничения: бюджет <$2000, контент — только цифровой.
Формат вывода: краткий вывод + пошаговый план (10 шагов).
```

### Meta / OODA — пример заполнения

```
TOPIC = "оптимизация этого мастер-промта"
MAX_ITER = 3
EPSILON = 1.0
BRANCHES = "main,experimental"
```

## Инструкция по использованию (4 шага)

1. Открой новый чат.
2. Вставь выбранный блок (Quick / Standard / Meta).
3. Замени все `{PLACEHOLDERS}` на свои значения.
4. Отправь — и получи ответ. Для итеративного режима — повторяй с новым `topic` или проси «продолжай итерацию» (если ассистент поддерживает state).
