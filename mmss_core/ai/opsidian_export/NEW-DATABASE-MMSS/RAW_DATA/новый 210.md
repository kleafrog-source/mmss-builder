Ручной запуск
Здесь мы выбираем ручной запуск. Причина:

Автоматический режим больше подходит для ситуаций, когда все узлы потребляют меньше времени, например, для простых вычислительных узлов. Ручной режим больше подходит для решения сложных и трудоёмких задач.
В процессе редактирования может появиться неполный код, неполный URL и т. д. В этот момент легко допустить ошибки.
Ручное управление также может имитировать автоматическую работу, автоматически добавлять таймер или отслеживать запуск преобразования, и это все.
Версия только для отображения. Простейшая версия предназначена только для отображения. Благодаря 节点项центрированной стратегии построения пользователи могут настраивать различные узлы без программирования для отображения различных структур. Этот поток узлов используется для отображения и не требует запуска и реализации работающего кода.
Режим привода
Драйвер узла данных
В основном используется для материальных процессов и т. д.
Например: редактирование материалов в различных 3D-программах происходит таким образом, в Comfyui это тоже происходит таким образом.
Драйвер: Когда узел X имеет несколько восходящих потоков данных, узел X начинает работать после завершения всех восходящих потоков данных.
Особенности: Сокеты данных часто имеют только один вход, но узел может иметь несколько входных сокетов данных.
По сути, его можно рассматривать как специальный сокет процесса. Начальный узел автоматически определяет
Реализация: статус узла делится на: не запущен, запущен и завершен (продолжить, когда данные будут готовы).
Драйвер узла процесса
В основном используется для управления технологическими процессами и т. д.

Драйвер: После активации сокета входного процесса на узле X узел немедленно запускается

Драйвер гибридного узла
Например: UE такой
Драйвер: Сначала узел без сокета процесса автоматически запускается и завершает работу. Затем узел с сокетом процесса запускается с самого начала.
После запуска сокета входного процесса на узле X он также должен соответствовать другому условию: его сокет входных данных находится в состоянии продолжения.
То есть, узел X одновременно удовлетворяет условиям запуска как узла данных, так и узла процесса, и он будет автоматически запущен.
Особенности: Сокеты делятся на сокеты данных и сокеты процессов, каждый из которых имеет свои собственные характеристики.
Проблемы согласованности данных
Источник данных

Исходные данные (могут быть не в формате JSON, могут быть ненормализованными)
Самостоятельно поддерживаемые данные (JSON, напрямую сериализуются и десериализуются с исходными данными. С вложенными структурами)
Данные по обслуживанию библиотеки (содержат информацию библиотеки по умолчанию. Не имеют вложенной структуры. Обычно содержат объекты с самообслуживаемыми данными)
последовательность

Первое и второе значения обычно не имеют отношения к согласованности, они напрямую связаны посредством ручной сериализации и десериализации. Упомянутая здесь согласованность в основном касается согласованности между вторыми и третьими данными.
Нет необходимости беспокоиться о согласованности данных внутри узла, поскольку эта часть — третья, включающая в себя вторую.
Основные, на которые следует обратить внимание:
Первый основан на данных, для запуска которых требуется updateNodeData (например, isRunningстатус).
Второй — это добавление, удаление, идентификатор и местоположение. Эти данные, которых нет в узле, содержат содержимое.
