### 📝 **ЗАДАНИЕ ДЛЯ ИИ-АССИСТЕНТА (Windsurf SWE-1)**  
**Цель:** Доработать веб-приложение `SeedLib` до состояния удобной локальной системы каталогизации мета-формул, скриптов, чатов и команд с поддержкой связей, тегов, категорий и ИИ-наполнения.

---

#### 🔧 **1. НОВЫЕ ПОЛЯ В МОДЕЛИ `Version`**
Добавить в класс `Version` следующие поля:
- `category` — строка: одна из предопределённых категорий (`full_chats`, `script_versions`, `seeds`, `commands_catalogue`)
- `related_seeds` — список ID связанных элементов из категории `seeds` (массив строк)
- `user_influence_vector` — текстовое поле: "вектор влияния пользователя" (описание, как пользователь повлиял на создание)
- `meta_formula_tags` — список строк: теги мета-формул (например: `["Ψ149", "Ω7 ⊕ Φ9", "∇·S_context"]`)

> ⚠️ Все поля должны сохраняться в `metadata.yaml` и загружаться при `Version.load()`.

---

#### 🗂️ **2. УПРАВЛЕНИЕ КАТАЛОГАМИ (КАТЕГОРИЯМИ)**
- Добавить **отдельный JSON-файл** `/data/categories.json` для хранения списка категорий:
  ```json
  {
    "categories": [
      {"id": "full_chats", "name": "Полные чаты"},
      {"id": "script_versions", "name": "Версии скриптов"},
      {"id": "seeds", "name": "Seeds"},
      {"id": "commands_catalogue", "name": "Каталог команд"}
    ]
  }
  ```
- Реализовать API-эндпоинты:
  - `GET /api/categories` — возвращает список категорий
  - `POST /api/categories` — создаёт новую категорию
  - `DELETE /api/categories/<category_id>` — удаляет категорию (только если в ней нет элементов)

---

#### 🏷️ **3. ТЕГИ МЕТА-ФОРМУЛ**
- В UI добавить поле ввода для **мета-формул как тегов**:
  - При вводе текста и нажатии **Enter** — тег добавляется в список
  - Теги отображаются как **чипсы (chips)** с крестиком для удаления
  - Поддержка редактирования: можно удалять/добавлять теги
- Хранить как массив строк в `meta_formula_tags`

---

#### 🔗 **4. ВЗАИМОСВЯЗИ SEEDS ↔ SCRIPTS**
- В карточке элемента категории `seeds` добавить:
  - Список связанных элементов из `script_versions` (по `related_seeds`)
  - Кнопку **«Связать с версией»** → открывает модальное окно с выбором из списка скриптов
- В карточке `script_versions` — отображать связанные seeds (только для чтения)

---

#### 📋 **5. UI / UX УЛУЧШЕНИЯ**
- **Тёмная тема**: применить CSS, максимально близкий к **Open WebUI (rose-pine theme)**:
  - Фон: `#191724`
  - Акцент: `#ebbcba` (розовый), `#31748f` (голубой)
  - Текст: `#e0def4`
  - Карты: `#1f1d2e`
- **Кнопки Copy/Paste**:
  - Рядом с каждым текстовым полем (title, description, meta_formulas, content) — кнопки 📋 (copy) и 📥 (paste из буфера)
  - Реализовать через `navigator.clipboard`
- **Удаление**:
  - Кнопка 🗑️ в карточке и в деталях → подтверждение → `DELETE /api/versions/<id>`
- **Поиск**:
  - Поле поиска в шапке → ищет по `title`, `description`, `meta_formula_tags`, `content`
  - Обновляет список без перезагрузки

---

#### 🌐 **6. СТРУКТУРА СТАРТОВОГО ЭКРАНА**
На главной странице (`/`) отображать:
1. **Список категорий** (слева, как панель)
2. При клике на категорию — **раскрывать список элементов** в этой категории (только названия + дата)
3. При клике на элемент — **открывать его в правой панели** (как сейчас)
4. **Кнопка «Инструкции»** (в шапке) → модальное окно с текстом (см. ниже)
5. **Поисковая строка** (в шапке)

---

#### 📚 **7. ИНСТРУКЦИИ ДЛЯ ИИ-АССИСТЕНТА**
Добавить модальное окно с текстом:

> **Инструкция для ИИ-Ассистента (например, Perplexity Comet)**  
>  
> **Цель:** Наполнять библиотеку `SeedLib` данными из сессий DeepSeek, ChatGPT, Qwen, Grok.  
>  
> **Как работать:**  
> 1. Извлекай **полные чаты** → сохраняй в категорию **«Полные чаты»**.  
> 2. Выделяй **seed-запросы** (например: `"Ξ19 — seed тишины между словами"`) → сохраняй в **«Seeds»**.  
> 3. Для каждого seed:  
>    - Указывай **мета-формулу** как тег (например: `Ξ19 = ∫(∇·S_context) dt`)  
>    - Заполняй **вектор влияния**: как пользователь повлиял (например: «запрос на тишину»)  
> 4. Если есть **скрипты/версии кода** — сохраняй в **«Версии скриптов»** и **связывай с соответствующими seeds**.  
> 5. Полезные **одноразовые запросы** (например: «объясни квантовую запутанность») — в **«Каталог команд»**.  
>  
> **Важно:**  
> - Не дублируй данные.  
> - Используй точные названия категорий.  
> - Сохраняй контекст: дата, модель, суть диалога.

#### 🛠️ **8. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ**

- Сохранение — **атомарное** (никаких частичных записей)
- Поддержка **UTF-8** (все файлы)
- Совместимость с **Flask 2.3.3**
- Frontend — **чистый JS + Bootstrap 5** (без React/Vue)
