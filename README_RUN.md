# Инструкция по запуску MMSS System

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск веб-сервера

#### Вариант 1: Использование run.py (рекомендуется)
```bash
python run.py
```

#### Вариант 2: Прямой запуск Flask
```bash
python app.py
```

#### Вариант 3: Использование Flask CLI
```bash
flask run
```

### 3. Открытие в браузере

После запуска откройте в браузере:
- **http://localhost:5000**
- или **http://127.0.0.1:5000**

## Использование

### 1. Активация системы

1. Откройте главную страницу
2. Нажмите "Активировать MMSS"
3. Выберите домен применения
4. Нажмите "Активировать MMSS System"

### 2. Работа с модулями

#### PFR - Фрактальная Пересборка
- Перейдите в раздел "Модули" → "PFR"
- Заполните параметры задачи
- Нажмите "Выполнить фрактальную пересборку"

#### FRP - Темпоральная Навигация
- Перейдите в раздел "Модули" → "FRP"
- Настройте параметры сценария
- Нажмите "Начать темпоральную навигацию"

#### A-MMSS - Плетение Контекста
- Перейдите в раздел "Модули" → "A-MMSS"
- Заполните параметры первого и второго порядка
- Нажмите "Плетение контекста"

### 3. Дашборд

Перейдите в "Дашборд" для просмотра статуса всех компонентов системы.

## API Endpoints

### REST API доступны по следующим адресам:

- `POST /api/activate` - Активация системы
  ```json
  {
    "domain": "Финансовый Анализ"
  }
  ```

- `POST /api/pfr` - Выполнение PFR
  ```json
  {
    "domain": "Финансовый Анализ",
    "problem_data": {...}
  }
  ```

- `POST /api/frp` - Выполнение FRP
  ```json
  {
    "scenario_data": {...}
  }
  ```

- `POST /api/ammss` - Выполнение A-MMSS
  ```json
  {
    "context_data": {...}
  }
  ```

- `GET /status` - Статус системы

## Решение проблем

### Порт 5000 занят

Если порт 5000 занят, измените порт в `app.py` или `run.py`:

```python
app.run(debug=True, host='0.0.0.0', port=8080)  # Используйте другой порт
```

### Ошибка импорта модулей

Убедитесь, что все модули находятся в правильных директориях:
- `mmss_core/` должен содержать `__init__.py`
- `packages/` должна содержать все JSON файлы

### Проблемы с путями на Windows

Если возникают проблемы с путями, используйте абсолютные пути в `app.py`.

## Разработка

### Режим отладки

Приложение запускается в режиме отладки (`debug=True`), что позволяет:
- Автоматическую перезагрузку при изменении кода
- Детальные сообщения об ошибках
- Отладчик в браузере

### Логирование

Все операции логируются в консоль. Для продакшена рекомендуется настроить файловое логирование.

## Производственное развертывание

Для продакшена рекомендуется:

1. Использовать WSGI сервер (например, Gunicorn):
   ```bash
   pip install gunicorn
   gunicorn -w 4 -b 0.0.0.0:5000 app:app
   ```

2. Настроить Nginx как reverse proxy

3. Использовать переменные окружения для конфигурации:
   ```bash
   export FLASK_ENV=production
   export SECRET_KEY=your-secret-key
   ```

4. Отключить debug режим:
   ```python
   app.run(debug=False)
   ```
